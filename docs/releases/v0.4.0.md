# v0.4.0 Release Notes

## Summary

`v0.4.0` expands the platform from a Docker-only workstation launcher into a broader
container-and-VM control plane with dedicated admin and image-management surfaces.

This release adds the first working Proxmox VM adapter, unattended-install configuration for VM
sessions, reusable Proxmox image profiles, and browser-driven ISO lifecycle management.

## Highlights

- Added a dedicated Image Management page at `/images`
- Added a separate Admin Console at `/admin`
- Added the first working Proxmox VM provider adapter
- Added VM install-time configuration for locale, keyboard layout, timezone, and hostname
- Added reusable VM image profiles for unattended-install defaults and package selection
- Added Proxmox ISO import from URL, browser upload, listing, and deletion
- Added streamed large-file ISO upload handling through the control plane
- Added clearer ISO upload progress, refresh behavior, and `New` badges for recent uploads

## Included Changes

### User application

- Improved auth-first application flow
- Added provider-aware session launch
- Added VM session console access
- Added session metadata for Proxmox VM identity

### Admin console

- Separated admin functions from the user-facing session UI
- Added provider configuration controls
- Added cleanup operations and provider management improvements

### Image management

- Added Proxmox ISO inventory view
- Added ISO upload from local disk
- Added ISO import from remote URL
- Added ISO delete actions
- Added reusable VM image profile CRUD

### VM provisioning

- Added Proxmox-backed VM create/start lifecycle
- Added unattended installer seed generation and delivery
- Added launch-time install configuration inputs
- Added console handoff to Proxmox noVNC

### Platform/runtime

- Improved desktop runtime build feedback
- Improved installer behavior and repair flows
- Improved cleanup handling for stale backend state

## Known Limitations

- Large ISO upload UX is improved, but still has room for further polish during long-running finalization
- Proxmox VM console access currently relies on Proxmox console handoff rather than a gateway-proxied VM console
- Restart, hibernate, and resume lifecycle support are not yet implemented for VM sessions
- Local libvirt/KVM providers remain scaffolding only

## Upgrade Notes

- If you use Proxmox-backed VM launches, verify the configured provider has:
  - API connectivity
  - VM create/delete permissions
  - datastore permissions for ISO upload/delete on the configured ISO storage
- Refresh the Image Management page after upgrading to confirm current ISO inventory and image profiles
